    # 简单介绍下作用域？

    1. 作用域是指 变量 和 函数 可访问的范围。(自己理解：定义一个变量或者函数，能访问到它的范围就是作用域。)
    2. 作用域的作用：防止变量外泄，暴露；避免命名冲突。
    3. 作用域有全局作用域和函数作用域。全局作用域默认非严格模式情况下是window（node为global）；
    4. js的执行分为语法分析（语法分析有没有错误）、预编译（提升变量，函数声明）、解释执行三个阶段；
    5. 函数的预编译发生在函数执行的前一刻。
    6. 执行上下文（执行环境）：函数被执行的时候会创建执行环境（执行上下文），其中包含一个大对象，对象里存储着该执行环境的变量以及函数。这些大对象组成的链就是作用域链。

    # 介绍下原型链？js的继承？原型链与class的异同？

    # 原型

      每个函数（test）都会有prototype属性，指向该函数的原型对象（test.prototype），原型对象中有constructor属性，指向构造函数（test）。
  当创建该构造函数的实例时，实例对象会有__proto__属性，指向其构造函数的原型对象。

    # 原型链

      每个构造函数都有原型对象，原型对象都有只指向构造函数的指针，而每个实例都包含指向指向原型对象的内部指针；假如让一个原型对象等于另一个构造函数的实例，则这个原型对象就拥有了指向另一个构造函数原型对象的指针，如此层层递进，就形成了原型链。（也就形成了继承）

    # 介绍下构造函数是什么？执行new的过程？

    # 构造函数

      构造函数就是使用new关键字时调用的函数。

    # new的过程

      ·创建一个新对象；
  ·将构造函数的作用域赋给新对象（即this指向新对象）；
  ·执行构造函数中的代码（把构造函数中的属性添加到新对象身上）；
  ·返回这个新对象。

    # 事件循环？js事件委托（代理）？

    # 事件循环

    1.js是单线程；任务分为宏任务和微任务。
2.js执行的时候，先执行宏任务，宏任务又按照执行顺序分为同步任务和异步任务；
  同步任务指的是在JS引擎主线程上按顺序执行任务，执行完一个任务，再执行下一个任务，形成一个调用栈。
  异步任务不会进入主线程，会被推入到任务队列中，等待js主线程任务执行完后，去任务队列里面取任务执行。
3.js执行过程总结来说就是：
  ·执行宏任务中的同步任务，形成执行栈；
  ·把异步任务推到相应的管理线程中（例如：setTimeout->定时器触发线程）；
  ·异步任务满足条件后，会被推到任务队列中，等待主线程读取执行；
  ·主线程执行完以后，会检查是否存在微任务，若有，会执行完微任务后；
  ·当主线程的同步任务和微任务执行完后，会去读取任务队列，将里面的任务推到执行栈中执行（先进先出）；
  ·之后循环反复，直到任务全部执行完毕，就形成了事件循环。

    # 防抖，节流？

    # 防抖

    触发高频事件后n秒内函数只会执行一次，如果n秒内高频事件再次被触发，则重新计算时间；
思路：每次触发事件都取消之前的定时器。

       function debounce(fn){
    let timer = null
    return function(){
      timer && clearTimeout(timer)
      // 这里setTimeout里面的函数要使用箭头函数，如果使用普通函数的话，
      // 因为函数是在全局环境执行，test函数的this会指向window，即丢失this。
      // 或者可以添加代码let that = this,保存this
      // let that = this
      // timer = setTimeout(function(){
      //   fn.apply(that, arguments)
      // },1000)
      timer = setTimeout(() => {
        fn.apply(this, arguments)
      },1000)
    }
  }

    # 节流

    高频事件触发，但在n秒内只会执行一次，所以节流会稀释函数的执行频率；
思路：每次触发事件时都判断当前是否有等待执行的延时函数。

       function throttle(fn){
    let timer = null
    return function(){
      if(!timer) {
        console.log('timer', timer)
        timer = setTimeout(() => {
          timer = null
          fn.apply(this, arguments)
        }, 1000);
      }
    }
  }

    # ['1', '2', '3'].map(parseInt())?

    parseInt接收两个参数：转换目标，转换进制（基数）； radix为0时，且string参数不以“0x”和“0”开头时，按照10为基数处理。 map函数接收2个参数：callback函数，thisArg可选（执行 callback 函数时值被用作this）； map函数的callback函数有3个参数：当前元素，index，原数组； 所以：可以拆解为 ·parseInt(1，0) 1 ·parseInt(2，1) NAN ·parseInt(3，2) NAN

    # Set、 Map、 WeakSet 和 WeakMap 的区别？

    集合与字典的区别是：集合 是以 [value, value]的形式储存元素，字典 是以 [key, value] 的形式储存

    # 集合(Set)

    ·成员具有唯一性；
·只有键值，没有建名；
·可遍历，类似于数组;
·属性有size，constructor(构造函数)；
·方法有：add()、delete()、has()、clear()。

    # WeakSet

    ·属性有size，constructor(构造函数)；
·方法有：add()、delete()、has()；(clear被废弃。)
·只能存储对象引用，不能存储值；
·WeakSet 对象中储存的对象值都是被弱引用的，即垃圾回收机制不考虑 WeakSet 对该对象的应用，如果没有其他的变量或属性引用这个对象值，则这个对象将会被垃圾回收掉（不考虑该对象还存在于 WeakSet 中），所以，WeakSet 对象里有多少个成员元素，取决于垃圾回收机制有没有运行，运行前后成员个数可能不一致，遍历结束之后，有的成员可能取不到了（被垃圾回收了），WeakSet 对象是无法被遍历的（ES6 规定 WeakSet 不可遍历），也没有办法拿到它包含的所有元素

    # 字典(Map)

    ·任何具有 Iterator 接口、且每个成员都是一个双元素的数组的数据结构都可以当作Map构造函数的参数

    const set = new Set([
 ['foo', 1],
 ['bar', 2]
]);
const m1 = new Map(set);
m1.get('foo') // 1

const m2 = new Map([['baz', 3]]);
const m3 = new Map(m2);
m3.get('baz') // 3

    ·只有对同一个对象的引用，Map 结构才将其视为同一个键；（对象的地址不同，所以即使同名也不被看作是相同的）

     const map = new Map();

 map.set(['a'], 555);
 map.get(['a']) // undefined

    ·属性有：constructor、size
·方法有：set()、get()、has()、delete()、clear()

    # WeakMap

    # 注意，WeakMap 弱引用的只是键名，而不是键值。键值依然是正常引用

    ·WeakMap 对象是一组键值对的集合，其中的键是弱引用对象，而值可以是任意。
·WeakMap 中，每个键对自己所引用对象的引用都是弱引用，在没有其他引用和该键引用同一对象，这个对象将会被垃圾回收（相应的key则变成无效的），所以，WeakMap 的 key 是不可枚举的。
·方法有set()、get()、has()、delete()
·属性有：constructor

    # 总结

    # Set

    成员唯一、无序且不重复
[value, value]，键值与键名是一致的（或者说只有键值，没有键名）
可以遍历，方法有：add、delete、has

    # WeakSet

    成员都是对象
成员都是弱引用，可以被垃圾回收机制回收，可以用来保存DOM节点，不容易造成内存泄漏
不能遍历，方法有add、delete、has

    # Map

    本质上是键值对的集合，类似集合
可以遍历，方法很多可以跟各种数据格式转换

    # WeakMap

    只接受对象作为键名（null除外），不接受其他类型的值作为键名
键名是弱引用，键值可以是任意的，键名所指向的对象可以被垃圾回收，此时键名是无效的
不能遍历，方法有get、set、has、delete

    # Object.prototype.toString.call() 、 instanceof 以及 Array.isArray(),typeof的区别？

    # Object.prototype.toString.call()

    继承 Object 的对象都有 toString 方法，如果 toString 方法没有重写的话，会返回 [Object type]，其中 type 为对象的类型。但当除了 Object 类型的对象外，其他类型直接使用 toString 方法时，会直接返回都是内容的字符串，所以我们需要使用call或者apply方法来将Object原型上的toString方法作用在传入的obj的上下文中。

    # instanceof

    ·只能用来判断引用类型，不能判断基本类型；
·内部机制是判断类型的prototype是否出现在对象的原型链中。

    # Array.isArray()

    当判断是否为数组实例时，Array.isArray 优于 instanceof，因为前者可以检测 iframes。

    # typeof

    只能判断准确判断 基本类型 和 function。
